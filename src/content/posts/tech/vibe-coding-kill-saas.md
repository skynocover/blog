Vibe Coding 會讓 SaaS 消失嗎?

> 「SaaS 已死」

這句話的邏輯是這樣的：
現在的 vibe coding 已經**讓任何人都可以寫程式了**
誰還需要付月費買 SaaS？

前陣子我常用的 Markdown 線上工具一直壞
所以後來我乾脆直接叫 Claude code 寫一個給我
從頭到尾**花不到十分鐘，只下了三次 Prompt**
我就有了一個永遠不會壞掉、只專屬於我的 Markdown 工具
而且我想怎麼改就怎麼改 完全符合我用

這也是大家說 SaaS 已死的原因
當你可以做一個專屬於你的工具
你真的需要 SaaS 嗎?

SaaS 的本質是 Service 而不是 Software
SaaS 全名是 Software as a Service
重點在後面那個字：Service

**你買的不是軟體，是服務**

你訂閱 Notion
看上的不是 Notion 的程式碼
而是線上資料庫

你用 AWS、GCP 而不自己架伺服器
不是因為你架不起來
是因為上面有人 24 小時輪班幫你顧機器

你找律師問法律問題
不是因為他比你會查法條
而是因為你相信法律的問題他會處理得比你好

你找會計師記帳
不是因為你不會
是因為出事的時候他可以幫你負責

SaaS 的邏輯完全一樣
你付月費給 SaaS 公司
不只是買他的功能

你買的是：

半夜系統掛了，有人幫你修
資安出問題，有人幫你扛
資料庫改版了，有人幫你更新
**SaaS 賣的從來不是程式碼，是責任。**

這就延伸到了 SaaS 實際上在賣的第二個東西：持續的維運

軟體真正難的不是寫出來，而是怎麼維持
大家都在講 vibe coding 多快多厲害
一個禮拜就能做出一個 app

但**軟體真正的成本從來不在開發
而是在後續的營運**

為了節省 SaaS 訂閱費
你找了兩個工程師專門幫你們做公司內部的客製化軟體
一個工程師月薪 6 萬，兩個就 12 萬
你的 SaaS 訂閱費有到 12 萬嗎？

你說那不要找工程師了
讓原本的員工去學 Vibe coding 可以吧?
但員工的本業不是寫程式
多花時間做軟體，少花時間做本來的工作
真的有比較划算嗎?

而且**人都會離職，離職了，誰接手?**
接手的人有辦法只靠 AI 就繼續往下做嗎?


你說我是老闆，我自己來學總可以了吧? 我不會離職
你確定你有空嗎?
我遇過的案例都是
老闆要求全公司導入 ERP
結果老闆太忙了，沒時間學
變成老闆是全公司唯一不會ERP的人

這樣看來，Vibe coding 真的可以取代 SaaS 嗎?

**歷史從來不是取代，是疊加**
「X 會消滅 Y」這個句型，歷史上出現過太多次了

電視機出來的時候，大家說電影已死
結果電影院靠大螢幕、杜比音效變成體驗經濟
電影沒死，電視也發展成串流巨獸

PC 出現的時候，大家說大型主機要死了
結果 PC 從不到 1 億台長到近 10 億台
大型主機不但沒死，數據中心反而更大了

線上零售出現時，大家說實體店要完了
結果 Amazon 跟 Walmart 現在都是市值兆元級的公司
用完全不同的方式主導零售市場

每次的規律都一樣：

**新技術出現 → 有人說舊的要死 → 結果共存 → 總量變更多**

「vibe coding 會殺死 SaaS」跟當年「PC 會殺死大型主機」是同一個句型

Vibe coding 跟 SaaS 將會並存
Vibe coding 會把軟體這塊餅做大
讓軟體觸及到以前到不了的角落

原本做起來太花時間的事情，現在都可以做了
但這不代表 SaaS 會消失

對個人工作者來說，vibe coding 很有用
我以前沒自己做 Markdown 工具，就是因為時間成本不符
所以我只能忍受一天到晚壞掉的工具
當成本降低時，我終於可以解決一直卡在我心頭上的事

對大公司也有用
像 Tesla 這種本來就有軟體部門的垂直整合公司
他們本來就有專門的部門做工具給自己公司用
現在用Vibe coding可以更快做出來

但中小企業呢？
養不起軟體部門的公司
誰來做？員工？
員工離職了呢？
老闆？老闆有空嗎？

這些公司不但不會因為 vibe coding 離開 SaaS
反而**更需要SaaS**
因為他們沒有能力自己維運 Vibe coding 做出來的服務

反過來想，AI 以後有機會能自動維運嗎？
可以
但要有人告訴它該處理什麼
中小企業沒有這個「人」

而且再退一步想
就算 AI agent 真的能幫你全自動維運
你還是得付錢給提供這個 AI agent 的服務商
**那不就是另一種 SaaS？**

**SaaS 沒有生病，它只是被誤診了**
1. Vibe coding 改變的是「誰寫程式」，沒有改變「誰扛責任」

**寫出來從來不是最難的，扛下去才是。**

2. 共存才是常態

Vibe coding 將會把餅做大，讓軟體延伸到更多角落

3. 願意為你負責的人，從來都是稀缺的

那個人可以是員工，可以是老闆
但何不外包給 SaaS
讓你專心在你的本業
